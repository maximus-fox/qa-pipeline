0. **Browser driver — READ FIRST. You MUST drive a real browser and capture real screenshots. NEVER degrade to "API-only / blocked / ⚠️ВИЗУАЛ НЕ ПОКРЫТ" because a tool seems missing — you always have Bash, which is enough.** Two ways; use whichever works:
   - **(A) Playwright MCP** (`mcp__playwright__*`): try it first. If the tools are deferred, load them with `ToolSearch` (query `playwright`). In MANY environments this MCP is "Connected" yet NON-FUNCTIONAL — its bundled browser isn't installed, or the tools don't resolve and calls return *"No such tool available"*. If that happens, DROP it immediately — do not retry it, do not conclude the run is blocked — and switch to (B).
   - **(B) Playwright via Bash — the reliable default (this is how the run actually gets done).** Bash is always in your toolset, and the `playwright` npm package + system Chrome are present, so this works even when (A) doesn't. Pattern (macOS):
     ```bash
     # locate the playwright module — npx cache dir is hash-named & non-deterministic, so DETECT it (don't hardcode the hash)
     PWPATH="$(dirname "$(ls -d /Users/max/.npm/_npx/*/node_modules/playwright 2>/dev/null | head -1)")"
     [ -z "$PWPATH" ] && npm i --prefix /tmp/pw playwright >/dev/null 2>&1 && PWPATH=/tmp/pw/node_modules   # fallback: install once
     NODE_PATH="$PWPATH" node /tmp/qa-driver.mjs <url> <w>x<h> <out.jpg>
     ```
     In the Node script: `const {chromium}=require("playwright"); const b=await chromium.launch({executablePath:"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"});` — **always pass `executablePath` to the system Chrome; the bundled chromium is usually absent.** Per page: `const p=await b.newPage({viewport:{width,height}}); p.on('console',m=>…); p.on('response',r=>…)` to capture console/network; `await p.screenshot({path:'<run-folder>/<screen>-<viewport>.jpg'})`. Use the test account's real logged-in session (set cookies / log in via the form); only `page.route('**/api/**', …)`-stub if a backend genuinely isn't reachable. **Write ONE reusable driver to `/tmp` and call it per page/flow** (args: url, viewport, out-path, steps) instead of rewriting boilerplate. Each `node` run launches its OWN Chrome — so multiple testers in parallel never contend for a single browser. **If Node's `playwright` won't resolve** (npx cache dir without it, cwd issues), the **Python** Playwright (`pip`'s — usually on PATH with its OWN working bundled Chromium) is an equally valid driver: `python3 - <<'PY'\nfrom playwright.sync_api import sync_playwright\nwith sync_playwright() as pw:\n  b=pw.chromium.launch()  # bundled Chromium; add channel="chrome" to force system Chrome\n  p=b.new_page(viewport={"width":1440,"height":900})  # [ТОЛЬКО FALLBACK] — подставь реальные viewport-ы из иерархии выше\n  p.goto(url); p.screenshot(path=out)\nPY`. Use whichever of Node/Python resolves first — both produce identical screenshots. **Smoke-test the recipe once at the start** (`NODE_PATH="$PWPATH" node -e 'require("playwright")'` + one screenshot) before walking the matrix; if even (B) fails, say exactly why (e.g. Chrome missing) — that is the ONLY legitimate path to a "visual not covered" outcome.

Если brief называет только часть — тестируй только их. ЕСЛИ цель — **Mini App** (Telegram, MAX или иной мессенджер): тестируй веб-приложение напрямую, но учитывай, что контекст мессенджера (`initData`, SDK-объект) в голом браузере воспроизвести нельзя — отсутствие SDK-объекта не является багом; используй cookie/email-сессию приложения, если она есть. Save every screenshot with an absolute path inside the run folder, **suffixed with the viewport** (e.g. `04-dashboard-mobile.jpg`). You have no Write tool — screenshots written by the Bash driver are how artifacts persist.
   **Screenshot protocol (so the visual axis is actually covered):** before each capture — kill motion (`await p.addStyleTag({content:'*{animation:none!important;transition:none!important;scroll-behavior:auto!important}'})`), `await p.evaluate(()=>document.fonts.ready)`, wait for network-idle / the key element; JPEG (not heavy PNG); generous timeout. Capture **one clean full screen per key screen per viewport** — the raw material for the visual critique. If a capture still fails after one retry, record it and **headline the Summary «⚠️ ВИЗУАЛ НЕ ПОКРЫТ»** — but reaching that headline is legitimate ONLY after driver (B) genuinely failed, NEVER as a shortcut for "the MCP tool wasn't there".
